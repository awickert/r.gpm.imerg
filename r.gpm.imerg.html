DESCRIPTION

r.gpm.imerg downloads NASA GPM IMERG (Integrated Multi-satellitE Retrievals for GPM) global precipitation data and imports raster maps into GRASS GIS.

IMERG provides global precipitation estimates at 0.1° spatial resolution from 60°S to 60°N, covering 2000-06-01 to near-present. Data are accessed via NASA Earthdata using the earthaccess Python library; a free NASA Earthdata account is required.

Each granule is imported as a separate raster map named <output>_YYYYMMDD (daily), <output>_YYYYMM (monthly), or <output>_YYYYMMDD_HHMM (half-hourly), and automatically reprojected and clipped to the current GRASS computational region via r.import.

NOTES

Products and processing runs

ProductCodeTemporal resolutionAvailable runs
Half-hourlyHH30 minutesE, L, F
DailyD1 dayE, L, F
MonthlyM1 monthF only

Processing runs differ in latency and accuracy:

RunCodeLatencyNotes
EarlyE~4 hoursNear-real-time; least accurate
LateL~12 hoursNear-real-time; improved over Early
FinalF~3.5 months Research quality; gauge-corrected

Variables

CodeDescriptionUnits
precipitationCal Calibrated precipitation (gauge-corrected for Final run) mm/hr
precipitationUncal Uncalibrated multi-satellite precipitation mm/hr
HQprecipitation High-quality microwave-only precipitation mm/hr
IRprecipitation Infrared-only precipitation mm/hr
randomError Random error estimate mm/hr
probabilityLiquidPrecipitation Probability of liquid (vs. solid) precipitation %

All precipitation variables are stored as rates (mm/hr), not accumulations. To convert daily output to a daily total: 

r.mapcalc "pr_mm_day = pr_20200101 * 24"

For half-hourly output, multiply by 0.5 to get mm per 30-minute period.

Authentication

A free NASA Earthdata account is required. Credentials are read in order:
  1. Environment variables EARTHDATA_USERNAME and EARTHDATA_PASSWORD
  2. ~/.netrc entry for urs.earthdata.nasa.gov
  3. Interactive prompt (if running in a terminal)

Download strategy

r.gpm.imerg uses earthaccess to search the NASA Common Metadata Repository (CMR) and download matching granules to a temporary directory. Files are processed one at a time and deleted after import to minimize disk usage.

Half-hourly requests produce 48 granules per day (~5–10 MB each). For a multi-day request, consider using the daily product instead.

Spatial coverage

IMERG provides coverage from 60°S to 60°N. Pixels outside this range are fill values. r.import clips data to the current GRASS computational region, so regions outside ±60° latitude will be empty.

Space-time raster datasets

Use the -t flag to register imported maps as a GRASS space-time raster dataset (strds). Each map's temporal extent covers one granule period (30 minutes, 1 day, or 1 month).

EXAMPLES

Import daily Final-run calibrated precipitation for January 2020: 
r.gpm.imerg output=pr variable=precipitationCal product=D run=F \
    start=2020-01-01 end=2020-01-31
Import and register as a space-time raster dataset: 
r.gpm.imerg output=pr variable=precipitationCal product=D run=F \
    start=2020-01-01 end=2020-12-31 -t
Import a single day of half-hourly Early-run data: 
r.gpm.imerg output=pr_hh variable=precipitationCal product=HH run=E \
    start=2020-07-04 end=2020-07-04 -t
Import monthly Final-run precipitation for a full year: 
r.gpm.imerg output=pr_mo variable=precipitationCal product=M run=F \
    start=2020-01-01 end=2020-12-31 -t
Convert imported daily rate maps to daily totals (mm/day): 
for map in $(g.list type=raster pattern="pr_2020*"); do
    r.mapcalc "${map}_mm = ${map} * 24" --overwrite
done

REQUIREMENTS

  • Python package earthaccess — pip3 install --break-system-packages earthaccess
  • Python package h5py — sudo apt install python3-h5py (Debian/Ubuntu)
  • GDAL CLI tools (gdal_translate) — typically installed with GRASS
  • NASA Earthdata account — urs.earthdata.nasa.gov
